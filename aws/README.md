# CARE Compass AWS Infrastructure

Serverless deployment for CARE Compass on AWS Lambda + API Gateway.

## Architecture

```
Electron Frontend
    ↓ (HTTPS)
API Gateway → Lambda (care-compass-dev)
    ├── ChromaDB (restored from S3 on cold start)
    ├── Google Gemini AI (LLM + Embeddings)
    ├── DynamoDB (conversation history)
    ├── Supabase (JWT authentication)
    └── S3 (vectorstore backup + interaction logs)
```

## File Structure

```
aws/
├── lambda/
│   ├── lambda_handler.py    # Lambda handler (session_id + JWT auth)
│   ├── s3_manager.py        # S3 integration
│   ├── requirements.txt     # Lambda Python dependencies
│   └── build/               # Deployment package (generated by deploy.ps1)
├── infrastructure/
│   └── template.yaml        # CloudFormation template (Lambda, API GW, DynamoDB, IAM)
├── deployment/
│   └── deploy.ps1           # PowerShell deployment script (AWS CLI only, no SAM needed)
├── scripts/
│   └── upload_vectorstore.py # Upload local vectorstore to S3 for Lambda
└── README.md
```

## Prerequisites

- AWS CLI v2 (no SAM CLI needed)
- Python 3.11+
- AWS IAM user with permissions: CloudFormation, Lambda, API Gateway, S3, DynamoDB, IAM
- Google Gemini API key
- Supabase JWT secret
- Vectorstore built locally (run backend once to generate it)

## Deploy

### 1. Upload vectorstore to S3

The vectorstore must be built locally first (start the backend, it auto-builds on first run). Then upload:

```bash
python aws/scripts/upload_vectorstore.py
```

This zips `data/processed/vectorstore/` and uploads to `s3://care-compass-vectordb-<account>-dev/vectorstore/vectorstore.zip`.

### 2. Configure AWS profile

If deploying to a different AWS account than your default:

```bash
aws configure --profile carecompass
```

### 3. Deploy

```powershell
.\aws\deployment\deploy.ps1 `
    -Environment dev `
    -AWSProfile carecompass `
    -GoogleApiKey "your_gemini_key" `
    -SupabaseJwtSecret "your_supabase_jwt_secret"
```

This will:
- Build Lambda package (copy code + install pip deps + remove bloat)
- Package and upload to S3
- Deploy CloudFormation stack (Lambda, API Gateway, DynamoDB, IAM)
- Output the API Gateway URL

### 4. Update frontend

Point your Electron frontend to the API Gateway URL from the deploy output.

## What Gets Created

| Resource | Name | Purpose |
|----------|------|---------|
| Lambda Function | `care-compass-dev` | Runs the RAG chatbot |
| API Gateway | `care-compass-api-dev` | HTTPS endpoints |
| DynamoDB Table | `care-compass-conversations-dev` | Conversation history (TTL-enabled) |
| IAM Role + Policies | | Lambda execution with S3 + DynamoDB access |

**Pre-existing S3 buckets** (not managed by CloudFormation):
| Bucket | Purpose |
|--------|---------|
| `care-compass-documents-*-dev` | Raw source PDFs |
| `care-compass-vectordb-*-dev` | Vectorstore backup (.zip) |
| `care-compass-logs-*-dev` | API interaction audit logs |

## Lambda Configuration

- **Runtime:** Python 3.11
- **Memory:** 512 MB (no ML models, API-only)
- **Timeout:** 300s
- **Ephemeral Storage:** 1 GB (for ChromaDB in /tmp)
- **Cold start:** Downloads vectorstore.zip from S3, extracts to /tmp
- **Warm invocations:** Reuses /tmp (no re-download)

## Updating the Vectorstore

1. Add/modify PDFs in `data/raw/`
2. Delete `data/processed/vectorstore/` and restart the backend locally (rebuilds via Gemini API)
3. Re-upload: `python aws/scripts/upload_vectorstore.py`
4. Lambda picks up the new vectorstore on next cold start (or redeploy to force)

## API Endpoints

| Endpoint | Method | Auth | Purpose |
|----------|--------|------|---------|
| `/chat` | POST | Bearer token or session_id in body | Send query, get response |
| `/clear` | POST | Bearer token or X-Session-ID header | Clear conversation history |
| `/health` | GET | None | Health check |
| `/stats` | GET | None | Bot statistics |
| `/categories` | GET | None | Available scenarios |

## Useful Commands

```powershell
# View Lambda logs
aws logs tail /aws/lambda/care-compass-dev --follow --profile carecompass

# Check stack status
aws cloudformation describe-stacks --stack-name care-compass-stack-dev --profile carecompass --query 'Stacks[0].StackStatus'

# List S3 interaction logs
aws s3 ls s3://care-compass-logs-432732422396-dev/interactions/ --recursive --profile carecompass

# Test health endpoint
curl https://<api-id>.execute-api.us-east-1.amazonaws.com/dev/health
```

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Lambda package too large (>250MB) | deploy.ps1 removes bloat automatically; check if new deps were added |
| Stack creation fails (bucket exists) | S3 buckets are pre-existing; template references them by name, doesn't create them |
| Lambda timeout | Increase in template.yaml `LambdaTimeout` or deploy with `-LambdaTimeout 900` |
| Vectorstore empty on Lambda | Run `python aws/scripts/upload_vectorstore.py` to upload to S3 |
| Permission denied | IAM user needs CloudFormation, Lambda, S3, DynamoDB, IAM, API Gateway permissions |

## Cleanup

```bash
# Delete the CloudFormation stack (keeps S3 buckets)
aws cloudformation delete-stack --stack-name care-compass-stack-dev --profile carecompass
```
