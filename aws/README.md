# CARE Compass AWS Infrastructure

Serverless deployment for CARE Compass on AWS Lambda + API Gateway.

## Architecture

```
Electron Frontend
    ↓ (HTTPS)
API Gateway → Lambda (care-compass-dev)
    ├── ChromaDB (restored from S3 on cold start)
    ├── Google Gemini AI (LLM + Embeddings)
    ├── DynamoDB (conversation history)
    ├── Supabase (JWT authentication)
    └── S3 (vectorstore backup + interaction logs)
```

## File Structure

```
aws/
├── lambda/
│   ├── lambda_handler.py    # Lambda handler (session_id + JWT auth)
│   ├── s3_manager.py        # S3 integration
│   ├── requirements.txt     # Lambda Python dependencies
│   └── build/               # Deployment package (generated by deploy.ps1)
├── infrastructure/
│   └── template.yaml        # CloudFormation template (Lambda, API GW, DynamoDB, IAM)
├── deployment/
│   └── deploy.ps1           # PowerShell deployment script (AWS CLI only, no SAM needed)
├── scripts/
│   └── upload_vectorstore.py # Upload local vectorstore to S3 for Lambda
└── README.md
```

## Prerequisites

- AWS CLI v2 (no SAM CLI needed)
- Python 3.11+
- AWS IAM user with permissions: CloudFormation, Lambda, API Gateway, S3, DynamoDB, IAM
- Google Gemini API key
- Supabase JWT secret
- Vectorstore built locally (run backend once to generate it)

## Deploy

### 1. Upload vectorstore to S3

The vectorstore must be built locally first (start the backend, it auto-builds on first run). Then upload:

```bash
python aws/scripts/upload_vectorstore.py
```

This zips `data/processed/vectorstore/` and uploads to `s3://care-compass-vectordb-<account>-dev/vectorstore/vectorstore.zip`.

### 2. Configure AWS profile

If deploying to a different AWS account than your default:

```bash
aws configure --profile carecompass
```

### 3. Deploy

```powershell
.\aws\deployment\deploy.ps1 `
    -Environment dev `
    -AWSProfile carecompass `
    -GoogleApiKey "your_gemini_key" `
    -SupabaseJwtSecret "your_supabase_jwt_secret"
```

This will:
- Build Lambda package (copy code + install pip deps + remove bloat)
- Package and upload to S3
- Deploy CloudFormation stack (Lambda, API Gateway, DynamoDB, IAM)
- Output the API Gateway URL

### 4. Update frontend

Point your Electron frontend to the API Gateway URL from the deploy output.

## API Endpoints

| Endpoint | Method | Auth | Purpose |
|----------|--------|------|---------|
| `/chat` | POST | Bearer token or session_id in body | Send query, get response |
| `/clear` | POST | Bearer token or X-Session-ID header | Clear conversation history |
| `/health` | GET | None | Health check |
| `/stats` | GET | None | Bot statistics |
| `/categories` | GET | None | Available scenarios |

### Request & Response

**Chat request:**
```json
{"query": "What is trauma-informed care?", "session_id": "guest-abc123", "scenario": "mental_health"}
```
- `query` (required): User's message
- `session_id` (optional): Unique session ID for guest users. Not needed if using a JWT.
- `scenario` (optional): One of `immediate_followup`, `mental_health`, `practical_social`, `legal_advocacy`, `delayed_ambivalent`

**Chat response:**
```json
{
  "response": "Bot's reply text here...",
  "is_crisis": false,
  "num_docs_retrieved": 3,
  "scenario": null,
  "blocked": false
}
```

| Field | Type | Description |
|-------|------|-------------|
| `response` | string | The bot's reply |
| `is_crisis` | boolean | `true` if crisis language was detected (keywords or AI assessment) |
| `num_docs_retrieved` | number | How many reference documents were used for context |
| `scenario` | string/null | The scenario category used, if any |
| `blocked` | boolean | `true` if the response was blocked by safety filters |

### Testing with curl

```bash
# Health check
curl https://7i51eh59p0.execute-api.us-east-1.amazonaws.com/dev/health

# Chat
curl -X POST https://7i51eh59p0.execute-api.us-east-1.amazonaws.com/dev/chat \
  -H "Content-Type: application/json" \
  -d '{"query": "What is trauma-informed care?", "session_id": "test-123"}'

# Chat with auth
curl -X POST https://7i51eh59p0.execute-api.us-east-1.amazonaws.com/dev/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <jwt_token>" \
  -d '{"query": "How do I find a counselor?"}'

# Clear conversation
curl -X POST https://7i51eh59p0.execute-api.us-east-1.amazonaws.com/dev/clear \
  -H "X-Session-ID: test-123"

# Categories
curl https://7i51eh59p0.execute-api.us-east-1.amazonaws.com/dev/categories
```

> **Windows PowerShell:** Use `curl.exe` instead of `curl` and escape double quotes in JSON.


### Session Management

- **Guest users**: Generate a unique `session_id` on the frontend (e.g., `guest-<uuid>`) and send it with every request. History auto-expires after 30 minutes.
- **Authenticated users**: Send a Supabase JWT in the `Authorization` header. The backend extracts the user ID as the session ID.
